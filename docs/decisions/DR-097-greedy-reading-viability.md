# DR-097: 先食い・早閉じ抑制の「読める」の精密化 — 成立した読みへの選好

> 由来: DR-096 の bare long (`long_prefix: ""` 無条件合法化) 固定を経て、先食い規定 (DR-041 §4) の述語「読める」がトリガ一致を指すのか読みの成立を指すのかが `height height` (両トークンとも値照合失敗) のようなケースで問われた。kawaz 裁定 (2026-07-12) による確定。

## 決定

DR-041 §4 の先食い (および同型の早閉じ抑制) が使う述語「読める」を精密化する:

> 先食い (および同型の早閉じ抑制) の述語「読める」は**トリガ一致ではなく、読みの成立** — その greedy 読みを経由する完全経路が存在すること — を指す。greedy 読みが 1 本も完全経路に到達しない消費点では、トークンは読みゼロと同じ扱いになり素通し枝が生きる。そこから完全経路が立てば採用され、greedy 側の held Error は捨てられる (DR-037「他ルートが通るならエラーは捨てる」の適用)。

具体的には、ある消費点で greedy エントリのトリガがトークンに一致しても:

- その読み (トリガ発火 + 値スロット消費) を経由して §15.1 の完全解決経路 (入力を全消費し、遅延述語 §15.9 も含め全段が通る経路) が **1 本でも** 立つなら、先食いは維持される (素通し枝は生成されない、従来どおり)
- その読みを経由する完全経路が **1 本も** 立たない (値照合失敗・後続トークン枯渇などで held Error のみに終わる) なら、その消費点は読みゼロと同じ扱いになり、素通し (positional が raw で食う読み) の枝が生成される

### 論証: 既存教義の精密化であって新規則ではない

1. 先食いの目的 (DR-041 §4、垂直スライス PoC 根拠) は、greedy 読みと素通しの**両方**が完全経路になりうる場面での恒常 ambiguous を防ぐこと (anti-ambiguity の選好)。greedy 側の読みが1本も完全経路に到達しない場合、この目的は働かない — 抑制しても防ぐべき ambiguity は存在せず、唯一残りうる解釈 (素通し側) を黙って潰すだけになる。
2. DR-037 は「解けた (Accept 完了した) 枝の数」で結末を決め、経緯節で「途中エラーは評価を遅延して全ルートチェックして他が通るならエラーは捨てられる」ことを明文で確定している。トリガ一致だけを根拠に他ルート (素通し) の**生成自体**を封じる読み方は、この全ルートチェック教義と整合しない — 候補を生成前に間引くのではなく、全候補を生成してから成立可否で選ぶのが DR-037/038 の一貫した設計態度である。
3. DR-038 は「第2の完全経路を黙って潰すのは契約違反」とする (`-n1.0f` の greedy commit 非採用の論拠)。同型の違反が逆方向にも起こりうる: 成立しない読みのために唯一の完全経路 (素通し側) を黙って潰すのも同じ契約違反である。

これら3点から、「読める」の精密化は DR-037/038/041 の既存教義から演繹される帰結であり、新しい選好規則の追加ではない。

## 帰結 (fixture 固定)

kawaz 裁定 (2026-07-12) による確認例。定義: `config: {long_prefix: ""}`、options: `height` (number, long: true)、positionals: `rest` (string, repeat min 0)。

1. `height height` — 両トークンとも height の読みが成立しない (値照合失敗 / missing operand) → 両方素通し → `{rest: ["height", "height"]}` success
2. `height 168.5 height` — token1-2 は成立 (先食い維持) → `{height: 168.5}`。token3 は続く operand がなく不成立 → 素通し → `{height: 168.5, rest: ["height"]}` success
3. `height height 168.5` — token1 は値 "height" が number 照合失敗で不成立 → 素通しで rest へ。token2 は "168.5" で成立 (先食い) → `{height: 168.5, rest: ["height"]}` success

fixture: `fixtures/matcher-readings/long-empty-prefix-typed.json`。

## 不変な既存挙動

- `prog -v src` (`-v` は flag、値スロットを持たないため読みが常に成立=消費対象トークンなしで即座に完結) → src="-v" の読みは立たないまま。定義済み option と同形のリテラルを positional に渡す escape は従来どおり dd (`--`) が要る (DR-041 §4 の帰結は不変 — 変わるのは「読みが成立しない」場合だけ)
- dd (`--` は素の exact 一致で失敗しうる値照合を持たないため常に成立) の抑制は不変
- 受け皿 positional の無い定義では従来どおり failure + held Error 表示 (`fixtures/matcher-readings/cluster-split-no-flag.json` の `-n1.0f` 等)。精密化が効くのは素通し枝の**生成先** (受け皿) がある場合のみで、受け皿が無ければ held Error が唯一の結末になる点は変わらない — 親切なエラーは受け皿がある時だけ受容と引き換えに消える、という DR-041 受け入れる帰結 #1 (typo が positional に飲まれうる) と同じ向き
- `fixtures/path-search/held-errors-distinct-depth.json` / `held-errors-same-depth.json` / `fixtures/matcher-readings/prefix-guard-number.json` / `prefix-guard-string.json` は greedy vs 素通しの競合 (先食い規則の対象) を含まない (トリガ自体が不一致の未定義トークン、または or 枝競合) ため不変

## 早閉じ抑制への適用

DR-041 §4 は早閉じ抑制を「先食いの適用先を『素通し消費』から『スコープ退出』にも広げた同型規則」と定義しており、述語「読める」を共有する。したがって本 DR の精密化はそのまま及ぶ: スコープ内トリガがトークンに一致しても、その読みを経由する完全経路が成立しないなら、スコープ退出は妨げられない。本 DR では専用の fixture を追加せず、先食い側の fixture が同じ述語の精密化を証明していることをもって導出として明記するに留める。

## typo マスクの受容拡大

`height 16O.5` (typo、number 照合失敗) は、寛容な positional 受け皿がある定義では黙って rest に吸われ success になる — 現行の held Error 表示より受容側に倒れる。これは DR-041 §5 の prefix ガード非採用が既に受け入れた帰結 #1 (typo が positional に飲まれうる) の一貫適用であり、新しいトレードオフではない。緩和策も同じ: (a) 静的 warn (DR-021「option 群 + 上限なし string positional」構造の警告)、(b) 近接候補提示 (findings F-016)、(c) 方言ガードの opt-in。コアの既定は定義を信頼する。

## 採用しなかった案

### トリガ一致のままの現状維持

DR-041 §4 の字面 (トリガ一致 = 読める) を変えない案。値照合が failure する消費点でも先食いが素通し枝を封じ続けるため、唯一残りうる解釈を構造的に破壊する。DR-038 の「経路を黙って潰さない」契約に違反し、DR-096 の bare long (`long_prefix: ""`) 固定によって顕在化した `height height` のようなケースで唯一の妥当な読み (素通し) が失われる。

### 局所判定 (matcher 段の照合成立だけで判定)

「読める」を matcher 段の即時照合 (トリガ一致 + 値プリミティブの型照合まで) の成立可否で判定する案。全ルートチェック教義 (DR-037/038 の「経路全体を辿ってから結末を決める」) と一致せず、遅延述語 (§15.9、required 等) による下流失敗で唯一の完全経路が死ぬケースを取りこぼす非対称が残る — matcher 段では成立して見えても、後段の制約評価で完全経路になり損ねる場合に先食いが誤って発火し続ける。本 DR の「完全経路の成立」で判定する定式化はこの非対称を残さない。
